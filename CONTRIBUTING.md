# 🤝 Contributing to Kiro Professional Toolkit

We welcome contributions! This toolkit is designed to help developers maximize their productivity with Kiro IDE.

## 📋 How to Contribute

### 1. **Adding New Agents**
- Create JSON file in `agents/` folder
- Follow existing agent structure
- Include comprehensive system prompt
- Add to agents README.md

### 2. **Adding New Hooks**
- Create `.kiro.hook` file in `hooks/` folder
- Use descriptive names
- Test with different trigger types
- Document in hooks README.md

### 3. **Adding New Skills**
- Create `.md` file in `skills/` folder
- Include usage examples
- Provide complete implementation
- Update skills README.md

### 4. **Adding New Steering Rules**
- Create `.md` file in `steering/` folder
- Define clear file patterns
- Include practical examples
- Update steering README.md

### 5. **Improving MCP Configurations**
- Test MCP integrations thoroughly
- Document API key requirements
- Provide setup instructions
- Update MCP README.md

## 🔧 Development Guidelines

### Code Quality
- Follow existing patterns
- Include comprehensive documentation
- Test all functionality
- Use clear, descriptive names

### Documentation
- Update relevant README files
- Include usage examples
- Provide troubleshooting tips
- Keep documentation in sync

### Testing
- Test on clean Kiro installation
- Verify cross-platform compatibility
- Check all installation methods
- Validate with different project types

## 📝 Pull Request Process

1. **Fork the repository**
2. **Create feature branch**: `git checkout -b feature/amazing-feature`
3. **Make your changes**
4. **Test thoroughly**
5. **Update documentation**
6. **Commit changes**: `git commit -m 'Add amazing feature'`
7. **Push to branch**: `git push origin feature/amazing-feature`
8. **Open Pull Request**

## 🐛 Bug Reports

When reporting bugs:
1. Use clear, descriptive title
2. Describe expected vs actual behavior
3. Include steps to reproduce
4. Provide system information
5. Include relevant logs/screenshots

## 💡 Feature Requests

When requesting features:
1. Describe the use case
2. Explain why it's valuable
3. Provide implementation ideas
4. Consider existing alternatives
